# ADR-0003：识别明细库——统一持久化、JSON 单文件、落项目内同目录

日期：2026-06-16　状态：已采纳（issue #14）

## 背景

送货单识别结果只活在 `store.files`（纯 reactive，零持久化，[src/store.js:22](../../src/store.js)），关机/刷新即丢。「分供方详情」的合作工地/材料单价（`aggregateSupplierDelivery`）、「单价对比」（`aggregateItems`）都是**实时读 `store.files` 现算**，于是重启后全空。issue #14 要：把识别结果持久化，让两屏共读，重开仍在。

## 决策

1. **统一一个「识别明细库」**（CONTEXT 术语），「分供方详情」与「单价对比」两屏**共读同一份**持久数据 + 叠加当前会话 `store.files`，而非各写各的。

2. **权威存储用 JSON 单文件，落项目内同目录**（与 `分供方附件/` 同级），不进 localStorage。只存**结构化字段**：工地、公司(含来源层级)、日期、单号、材料明细行（名/单位/单价/数量/总价）、待复核标记；**不存原文件 blob**（blob 靠「导出归档」落 PDF）。
   - 否决 Excel 作权威：回读靠列名匹配（见 `importHistoryWorkbookItems`），会丢 **工地/单号/公司来源层级**，去重也难；Excel 是"给人看"的导出物，不兼任机读权威源。

3. **落盘机制沿用 ADR-0002**：dev 走 vite 中间件写项目内文件夹；打包桌面端走 Tauri Rust 文件命令（**待接**，与附件落盘同状态）。

4. **「保存(入库)」与「导出归档」解耦**（CONTEXT 术语）：前者写识别明细库；后者照旧出 公司文件夹 PDF + 工地汇总 Excel。两动作互不替代。

5. **去重/upsert 键 = 工地 + 公司 + 单号；单号必填**。
   - 入库时**有单号的正常 upsert，缺单号的本批扣下不入库**、单列出来等人补号再存（不因一张缺号拦整批）。「缺单号」由此成为一类硬性"不完善"项，与 #16 校验联动。
   - **覆盖核对**：key 命中且**新旧内容有变**才弹核对弹窗，列新旧对照、**逐条勾选**，默认**不勾**，勾中的才用新值覆盖、其余保留旧值；新旧完全一致则静默跳过；全新 key 静默入库。
   - 单号串进税号等脏值（CONTEXT「单号」条）属解析层问题，键只认 `doc.orderNo` 清洗后的值。

6. **每工地一个 JSON 文件**（项目内 `识别明细库/<工地>.json`，多文件同处一目录）：配合即将引入的「工地详情」页与 pinia 中央装配，按工地拆分便于工地级读写/备份；跨工地的「分供方详情」「单价对比」由 pinia 在内存汇总所有工地文件。键中的"工地"由所属文件隐含。
   - 注：本条**取代了初稿的"全局单文件"**——当 vue-router + pinia + 工地详情页 进入范围后改为按工地拆分（见 ADR-0004 / 重构 issue）。"无稳定每工地根目录"的顾虑由"全部文件同放项目内 `识别明细库/` 下"化解。

7. **未保存离开提醒**：有「已识别未入库」文件时，关窗/刷新前 `beforeunload` 提醒；「保存」按钮带未保存角标。切 tab 不提醒（store.files 单例不丢）。不做"识别完自动入库"（与显式保存+覆盖核对冲突）。

## 备选方案与否决理由

- **localStorage（沿用 supplierDb/manualGroups）**：浏览器+桌面立即可用、无"待接"。否决主因：用户要一份**可见、可直接备份、换机拷贝**的文件，契合本工具"按文件夹归档"的习惯；localStorage 不可见、换机不带走。代价是 production 落盘需补 Tauri 命令。
- **Excel(.xlsx) 作权威**：可双击核对，但作机读权威源会丢字段、回读脆弱、去重麻烦（见决策 2）。否决。
- **IndexedDB**：容量大、跨端统一，但不可见文件夹、迁移要另写导出，本工具桌面为主、用户偏好可见文件。否决（同 ADR-0002）。

## 影响

- production（Tauri）落盘**待接**，dev 先行可测；换机拷"项目目录"即带走库（与附件一致）。
- **单号升级为入库必填**：手写单缺号需人工补录才入库——多一道录入成本，换来稳定自然键与跨批次合并。
- 「分供方详情」的合作明细改为**按工地分组、瀑布流卡片**（日期/单号/采购明细）呈现，详情页从模态改为**独立标签页/路由**以去嵌套（见 issue #18，与 #12 排版优化交叠）。
