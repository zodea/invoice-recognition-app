# CONTEXT — 领域术语表

> 只收"这个项目里这些词到底指什么"，不放实现细节。冲突以本表为准。

## 通用

| 术语 | 含义 |
| --- | --- |
| **工地** | 一个施工项目（如"合和新城"）。送货单按工地归档：**每个工地一个 Excel**。应用内对应"分区(partition)"概念，两词同义，UI 用"工地/分区"。 |
| **分供方**（供应商） | 给工地供材料/服务的公司。资料含：公司全称、简称/别名、税号、**法人(法定代表人)**、**注册地址**、开户行、银行账号、联系人(签约代表)；**附件档案**：营业执照、法人身份证、银行开户许可证(＝基本存款账户信息)、签约代表身份证(正/反)、品牌授权证明(代理品牌时需)。**每个分供方一个工作表**（在所属工地的 Excel 里）。 |
| **简称 / 别名** | 分供方的短叫法（"大板东"＝"广州市大板东建材有限公司"）。来源：送货单子文件夹名、口头叫法。一个分供方可有多个别名；别名用于把文件夹名对应到全称。 |
| **送货单** | 分供方送材料时随货的单据（多为手写、盖红章扫描件）。关键字段：日期、单号、材料名称、单位、单价、数量、总价。正常都写日期。 |
| **单号** | 送货单编号，富丰这类预印单上是**红色流水号**（如 0005876）。常紧挨"统一社会信用代码/税号"印刷，扫描后 OCR 易把两者连成一串（如 `191206840950005876`，真正单号是尾部 `0005876`）。 |
| **多单判别（双联 vs 两单）** | 一个扫描文件里出现多张表/多张单时的判别规则：**单号不同 = 两张不同的送货单**（各自入账）；**单号相同 = 同一张单的双联/多联副本**（合并去重、不重复入账）。一张单常有多联复写（存根白/客户红/回单黄），归档以客户红联为准。 |
| **公司名以红章为准** | 送货单归属公司的判定规则：盖章名称优先；无章按抬头；再无按材料名临时分类并标待复核。 |
| **公司来源三层** | 公司字段的来源等级：**文件夹预填 < OCR识别 < 手动输入**。高层不被低层覆盖。任何两层**不一致都亮色标注**（"文件夹说A/识别是B"、"手填A/识别是B"），一键采用或保留，不一致即进待复核；一致则静默确认。 |
| **整理树** | 上传工地文件夹后的树状视图：工地 → 组（分供方或特殊组）→ 文件。可拖拽调整归属。**导入后也可重新打开调整；所有调整暂存在树里，点"确认"才生效**。工地层判定：默认最外层文件夹=工地，文件夹节点带**"是工地"复选框**可升/降级（应对拖入父文件夹、或第二层才是工地的情况）；根文件夹名若命中分供方简称则视为组并挂到"（未命名工地）"。支持手动**新建工地**空根。 |
| **特殊组** | 工地文件夹里不属于任何分供方的子文件夹（如"未签合同-送货单""住宅楼收据""专用账户管理协议"）。只是**入库前的中转分类**：组内文件照常识别，最终归属以识别出的公司为准；识别不出公司的，按特殊组名归类并标待复核。 |
| **公司文件夹** | 导出目录结构：工地文件夹 → **每个公司一个子文件夹**（公司=识别确认后的归属）→ 该公司的送货单 PDF。工地根目录放该工地的 Excel。 |
| **待复核** | 任何"机器不确定、需要人看一眼"的标记：日期缺失/被裁切、品名看不清、无章按材料分类、按文件名兜底补录等。汇总进 Excel 的"待复核清单"工作表。 |
| **识别明细库** | 送货单识别确认后、结构化明细的**持久留存**（跨工地累积，关机重开仍在）。是「分供方详情」的合作工地/材料单价、「单价对比」表的数据源——不再只靠当前会话现算。由**「保存到 分供方 & 单价对比」**动作写入。 |
| **保存(入库) vs 导出归档** | 送货单整理完成后的两个不同去向：**入库**＝把识别结果写进[识别明细库]，喂给分供方详情/单价对比；**导出归档**＝按公司文件夹出 PDF + 工地汇总 Excel。两者解耦，可只做其一，互不替代。 |
| **采购记录 vs 送货单(识别)** | 分供方详情对账里两类"进货"来源，**互不重叠以防重复计数**：**送货单(识别)**＝识别明细库里的送货单（带单号、按工地自动汇总）；**采购记录**＝**手动补记送货单之外的**采购（无单号，如口头/无单交易）。差额(应付)＝送货单合计 ＋ 手记采购 − 已支付。 |
| **单价对比·排除集** | 单价对比里"不按材料参与对比"的名称集合：默认**运费类关键词**（运费/运输费/送货费/搬运费/装卸费/配送费/上楼费/吊装费/卸车费）打底，用户可在"已剔除"清单里**加减、持久保存**；命中即从对比观测点剔除并**留痕**，下次同名自动剔除。仅作用于单价对比，**不影响**送货单明细与导出（运费仍是真实成本）。 |

## 发票侧

| 术语 | 含义 |
| --- | --- |
| **开票明细** | 逐张发票的字段记录：发票号码、开票日期、购/销方、金额、税额、价税合计、税点、类型。 |
| **汇总账单** | 按销售方汇总的金额/税额/价税合计统计（Excel 工作表）。 |
| **进项台账** | 历史进项发票清单（从税务局导出 Excel 导入），用于跨批次查重、记录已认证/已打印状态。 |
| **发票入账** | 发票经校对确认后进入已入账集合。入账发票的销售方视为一个分供方（供应商）候选，应纳入分供方资料库以便后续对账。 |
| **同类合并** | 同一公司的多份扫描文件合成一份 PDF（送货单侧）；发票侧指批量打印时拼版。 |
