# ADR-0002：分供方证照用本地 OCR 归类+回填，附件落本地文件系统

日期：2026-06-15　状态：已采纳

## 背景

分供方资料库（issue #6 / #7）要支持上传证照档案：营业执照、法人身份证、银行开户许可证（＝基本存款账户信息）、签约代表身份证（正/反）、品牌授权证明（代理）。用户要求：上传后**自动归类**并**回填表单字段**（公司全称、税号、法人、注册地址、开户行、银行账号、联系人）。

这些证照都是**规范印刷件**（固定版式、印刷体、关键词稳定），与手写送货单截然不同。

## 决策

1. **识别引擎用本地 PP-OCRv2（`public/ocr-models`），不用云 PaddleOCR-VL。** 证照是规范印刷件，本地小模型对印刷体足够（ADR-0001 实测：本地模型印刷体可用、仅手写差），从而**避免云 VL 的队列(code 10010)/成本/CORS** 问题、离线可用。流程：`renderFileToPages` → `recognizePages` → 关键词归类 → 按类抽字段回填。

2. **附件文件落本地文件系统（Tauri），不入 localStorage/Excel/IndexedDB。** 目录结构 `分供方/<公司全称>/<公司+类别>.<ext>`；根目录**默认应用数据目录，可在设置里自选其它根**。证照多为大图/PDF，localStorage 配额小、Excel 装不下；文件落盘无配额、用户可直接浏览/备份/换机拷贝，契合"按文件夹归档"习惯。分供方记录里只存 **相对路径 + 类别 + 原文件名**（仍在 localStorage）。

3. **回填规则**：只填**空字段**；与已填值不一致则亮色标「识别为 X，请核对」**不覆盖**；所有回填值可编辑（沿用"公司来源不一致"的人工确认思路）。

4. **浏览器 `npm run dev` 无文件系统**：上传降级为内存预览（object URL）仅供测试，不落盘；桌面端正常落盘。

5. 为容纳营业执照回填，分供方**新增字段：法人(法定代表人)、注册地址**。

## 备选方案与否决理由

- **云 VL 识别证照**：版式规范用不上 VL 的强手写能力，反受队列/成本/网络拖累。否决。
- **附件存 localStorage(base64) 或塞进 Excel**：配额与体积不可行（几张证照即超 localStorage ~5MB；Excel 无法承载二进制）。否决。
- **IndexedDB 存 blob**：浏览器+桌面统一、配额大，但迁移要另写"导出资料包"，且本工具桌面为主、用户偏好可见的文件夹。否决（保留为浏览器降级的可选项）。

## 影响

- 证照识别**离线可用、无云依赖**；本地模型对印刷体偶有错字，故回填值标「请核对」由人确认。
- 附件**不随 Excel 迁移**；换机需拷贝"资料根目录"（或后续加"一键导出资料包.zip"）。
- 新增分供方字段（法人、注册地址）需同步 Excel 导入/导出列。
- 「新增分供方」改为**弹窗**（reka-ui Dialog），不再用内联 div（避免触发列表重排）。
