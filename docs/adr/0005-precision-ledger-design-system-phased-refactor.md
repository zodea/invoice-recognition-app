# ADR-0005：采用 Precision Ledger 设计系统，分阶段重构 UI（视觉 → 壳 → 逐屏）

日期：2026-06-17　状态：已采纳（issue #12）

## 背景

#12（排版与界面优化）从"小打磨"升级为**整体重构**。用户提供 Stitch 设计系统参考 `stitch_invoice_recognition_design_system/`：
- `precision_ledger/DESIGN.md`：颜色/排版/组件规范（Corporate Modern、冷灰画布、卡片化、JetBrains Mono 数据字体、40/60 OCR 视图、紧凑表格）。
- `_1.._6/screen.png`：六张 mockup —— 仪表盘 / 发票识别校对(40-60) / 材料对比分析(图表) / 送货单识别校对(40-60) / 分供方库(KPI+环图) / 发票列表。
- 全局壳：**左侧边栏导航 + 顶部条**，取代现有顶部 Tab。

现有 `uno.config.ts` 的核心色（brand #2563eb、canvas #f5f6f8、card/边框、warning、brand-soft）**已与该系统对齐**（当初即照其搭建）。

## 决策

1. 以 **Precision Ledger** 为 #12 的视觉目标：**学样式与布局，保留本应用的范围（送货单+发票、中文）与功能**；不照搬 "InvoiceAI" 品牌名，也不照搬"发票独大"的信息结构。

2. **分阶段、先视觉后结构（方案 A，否决一次性大改）**：
   - **阶段① 视觉系统**：uno.config 扩全量 tokens + 字体（JetBrains Mono 本地打包用于数字/单号；Hanken Grotesk 仅影响拉丁字，中文仍用雅黑）+ 字阶/label-caps + 统一 btn/input/chip/table/upload 样式。全应用观感统一、**零工作流风险**。
   - **阶段② 壳**：顶部 Tab → 左侧边栏导航 + 顶部条（路由/激活态同步）。
   - **阶段③ 逐屏**：按 mockup 调各屏布局（仪表盘 / 40-60 校对 / 图表 / 分供方库）。
   - 每阶段独立提交、可回退。否决"一次性照 6 图重搭"（面大、风险高，易打乱现有 批量识别/整理树/保存入库 工作流）。

3. **送货单校对"逐单 vs 批量"留到该屏实现时再定**：现为"一屏多卡片批量 + 整理树 + 保存入库"，**倾向保留批量工作流、仅按设计系统重做卡片/明细样式**，而非改成 mockup 的逐单 40/60（除非用户明确要换工作流）。

## 影响

- 拆成多个 issue（视觉系统 / 壳 / 各屏）逐步落地，统一挂 #12。
- 新增字体资源（JetBrains Mono 本地打包，保离线）；图表页（仪表盘/单价对比）需新依赖（图表库，待选型）。
- 设计参考 `stitch_invoice_recognition_design_system/` 入库作为重构的单一事实来源（SSOT）。
- 仪表盘 KPI（总处理量/识别率等）需评估本地工具是否有真实数据来源，否则降级或缓做。
